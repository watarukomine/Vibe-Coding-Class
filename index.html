概念：バイブコーディング的 Webの仕組み

Webサイトは、大きく2つの要素でできています。これを人体に例えるとわかりやすいです。

  • HTML (HyperText Markup Language): 骨格・筋肉。「ここに画像」「ここに見出し」という構造を定義します。これがないと何も表示されません。
  • CSS (Cascading Style Sheets): 服・メイク。「文字を赤く」「右寄せに」という装飾を担当します。すっぴんでも機能はしますが、ユーザーを惹きつけるには必須です。

技術解説：DOMツリーの理解

AIが出力するコードでよく見る「入れ子構造」について。

<div class="container"> <h1>私の名前</h1> <p>自己紹介です</p> </div>

この div が「箱」で、その中に h1 (タイトル) や p (文章) が入っています。
「親要素のデザイン(CSS)は、子要素に遺伝することがある」
これを知っておくと、「なんでここが赤いの？」という謎が解けます。

AIハンドリング：修正のコツ

JSのキホン：変数と関数

AIコードを読むための最低限の知識。

  • const price = 1000;
    定数。中身を変えられない箱。「基本はこれを使う」と覚えればOK。
  • let total = 0;
    変数。中身を書き換えられる箱。計算結果を入れるときなどに使う。
  • const calculate = () => { ... }
    アロー関数。処理の塊。「この呪文を唱えると、波括弧の中身が実行される」イメージ。

Reactのキホン：State (状態)

「データが変われば、見た目も勝手に変わる」
これがjQueryなどの昔の技術とReactの決定的な違いです。

const [count, setCount] = useState(0);

1. count: 現在の数字（最初は0）
2. setCount: 数字を変えるための専用リモコン
ルール: count = 1 と直接書き換えてはダメ（反映されない）。必ず setCount(1) とリモコンを使うこと。そうすると画面がパッと書き換わる。

GASのオブジェクト指向

GASはGoogleのサービスを「モノ (Object)」として扱います。

  • GmailApp: メール機能全体をつかさどる神様。
  • SpreadsheetApp: 表計算機能の神様。
  • DriveApp: Googleドライブの神様。

GmailApp.sendEmail() のように、「神様.奇跡を起こせ()」と命令するイメージです。

トリガー：ボットの起動スイッチ

コードを書いただけでは動きません。「いつ」動くかを設定する必要があります。

  • シンプルトリガー: onEdit など。設定不要だが権限が弱い。
  • インストーラブルトリガー: 時計マークから設定。強力。「毎日朝9時」「フォーム送信時」などはこっち。今回はこれを使います。

トラブルシューティング：権限と承認

GAS初心者の9割がハマるのが「承認(Authorization)」の壁。

1. 初回実行時にポップアップが出る。
2. 「権限を確認」→ アカウント選択。
3. 「このアプリはGoogleにより確認されていません」と出る（怖い）。
4. 左下の「詳細」を押す → 「(安全ではないページ)に移動」を押す。
ここを突破しないと動きません。

Pythonの武器（ライブラリ）

Pythonが強いのは、天才たちが作った「道具箱」が無限にあるからです。

  • Requests: 人間の代わりにURLをクリックし、ページのデータを取ってくる道具。
  • BeautifulSoup: 取ってきたデータ（HTMLの闇鍋）から、欲しい情報（文字やお玉）だけを掬い取る道具。
  • Pandas: データ分析の王様。Excelの100倍速い表計算ソフトだと思ってください。

スクレイピングのマナー ⚠️

「DoS攻撃」にならないために。

  • Waitを入れる: time.sleep(1) などで、1秒に1回しかアクセスしないようにする。これ絶対。
  • robots.txt: サイトには「入っていい場所悪い場所」を書いた看板があります。
  • 利用規約: そもそもスクレイピング禁止のサイトもあります（Amazonなど厳しい）。今回は許可されている学習用サイトや、規約の範囲内で行います。

GitHub (ギットハブ) = セーブポイント

プログラマーのSNS兼、最強のバックアップ倉庫です。

  • Repository (リポジトリ): プロジェクトごとの「お道具箱」。
  • Commit (コミット): ゲームの「セーブ」と同じ。
    「ボス戦前(バグ修正前)の状態」などを保存しておけば、いつでも戻れます。
  • Push (プッシュ): 自分のPC(ローカル)から、ネット上のGitHub(リモート)へセーブデータをアップロードすること。

公開ステージ：Firebase vs Vercel

Webサイトの置き場所には、大きく2つの流派があります。

🔥 Firebase Hosting
コマンドラインでデプロイ。「エンジニアっぽさ」を味わいたい硬派向け。Google製で安心。
▲ Vercel
GitHub連携で全自動デプロイ。「爆速・手軽」を極めたい現代っ子向け。Next.js開発元が提供。

🐙 Step 1: GitHubへ保存

# 1. ここをGit管理下にする (初回のみ) git init git add . git commit -m "First vibe check" # 2. GitHubと繋ぐ (リポジトリURLは自分のものに！) git remote add origin https://github.com/ユーザー名/リポジトリ名.git git branch -M main git push -u origin main

※エラーが出たら「GitHub Desktop」などのGUIツールを使うのも手です。

🔥 Step 2: Firebaseで公開 (コマンド派)

Reactの場合は、まずビルドして「完成品」を作ります。

npm run build

次にFirebaseのツールを入れ、ログインします。

npm install -g firebase-tools firebase login

プロジェクトの初期化（これが一番大事）

firebase init hosting
質問への答え方 (React/Viteの場合)
1. Proceed? -> y
2. Option -> Create a new project (またはExisting)
3. Public directory? -> dist (超重要！ここを間違うと真っ白になります)
4. Single-page app? -> y
5. Automatic builds? -> n

準備ができたら、発射！

firebase deploy

▲ Step 3: Vercelで公開 (GUI派)

コマンド不要！ブラウザ操作だけで「自動公開」環境を作ります。

  1. Vercel公式サイトへ行き、Continue with GitHub でログイン。
  2. Add New... > Project をクリック。
  3. さっき作ったリポジトリ (`my-vibe-app`) を Import。
  4. 設定はいじらず Deploy ボタンをドーン！
✨ マジック：CI/CD
これ以降、手元のVS Codeで編集して git push するだけで、Vercel上の本番サイトも勝手に更新されるようになります。

概念：バイブコーディング的 Webの仕組み

Webサイトは、大きく2つの要素でできています。これを人体に例えるとわかりやすいです。

  • HTML (HyperText Markup Language): 骨格・筋肉。「ここに画像」「ここに見出し」という構造を定義します。これがないと何も表示されません。
  • CSS (Cascading Style Sheets): 服・メイク。「文字を赤く」「右寄せに」という装飾を担当します。すっぴんでも機能はしますが、ユーザーを惹きつけるには必須です。

技術解説：DOMツリーの理解

AIが出力するコードでよく見る「入れ子構造」について。

<div class="container"> <h1>私の名前</h1> <p>自己紹介です</p> </div>

この div が「箱」で、その中に h1 (タイトル) や p (文章) が入っています。
「親要素のデザイン(CSS)は、子要素に遺伝することがある」
これを知っておくと、「なんでここが赤いの？」という謎が解けます。

JSのキホン：変数と関数

AIコードを読むための最低限の知識。

  • const price = 1000;
    定数。中身を変えられない箱。「基本はこれを使う」と覚えればOK。
  • let total = 0;
    変数。中身を書き換えられる箱。計算結果を入れるときなどに使う。
  • const calculate = () => { ... }
    アロー関数。処理の塊。「この呪文を唱えると、波括弧の中身が実行される」イメージ。

Reactのキホン：State (状態)

「データが変われば、見た目も勝手に変わる」
これがjQueryなどの昔の技術とReactの決定的な違いです。

const [count, setCount] = useState(0);

1. count: 現在の数字（最初は0）
2. setCount: 数字を変えるための専用リモコン
ルール: count = 1 と直接書き換えてはダメ（反映されない）。必ず setCount(1) とリモコンを使うこと。そうすると画面がパッと書き換わる。

GASのオブジェクト指向

GASはGoogleのサービスを「モノ (Object)」として扱います。

トリガー：ボットの起動スイッチ

コードを書いただけでは動きません。「いつ」動くかを設定する必要があります。

  • シンプルトリガー: onEdit など。設定不要だが権限が弱い。

トラブルシューティング：権限と承認

GAS初心者の9割がハマるのが「承認(Authorization)」の壁。

1. 初回実行時にポップアップが出る。

3. 「このアプリはGoogleにより確認されていません」と出る（怖い）。
4. 左下の「詳細」を押す → 「(安全ではないページ)に移動」を押す。

Pythonの武器（ライブラリ）

Pythonが強いのは、天才たちが作った「道具箱」が無限にあるからです。

  • Requests: 人間の代わりにURLをクリックし、ページのデータを取ってくる道具。

スクレイピングのマナー ⚠️

「DoS攻撃」にならないために。

  • Waitを入れる: time.sleep(1) などで、1秒に1回しかアクセスしないようにする。これ絶対。
  • robots.txt: サイトには「入っていい場所悪い場所」を書いた看板があります。
  • 利用規約: そもそもスクレイピング禁止のサイトもあります（Amazonなど厳しい）。今回は許可されている学習用サイトや、規約の範囲内で行います。
